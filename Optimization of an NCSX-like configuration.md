Tutorial: STELLOPT Code
=======================

This tutorial will walk the user through running STELLOPT. For this
example the National Compact Stellarator Experiment (NCSX) configuration
will be used. This machine is stellarator symmetric with a periodicity
of three. The user should first run the
[VMEC Fixed Boundary Run](VMEC Fixed Boundary Run) tutorial to become
familiar with the basics of the VMEC code.

------------------------------------------------------------------------

# Edit the input namelist text file.

Just as with VMEC in input.<ext> file controls the execution of STELLOPT. In
 addition to the VMEC INDATA namelist, STELLOPT requires the OPTIMUM
 namelist (note that other namelists are included as well for various
 targets as necessary). It is advisable that the name of this file
 not contain the string '_opt' anywhere as this is used for
 temporary files which should be deleted at the end of a run. The
 VMEC2STEL tool can assist in generating such a namelist. For this
 example we provide the full input file below
 [input.ncsx_c09r00_fixed_stel](examples/input.ncsx_c09r00_fixed_stel).

# Examine the output.
A large part of the STELLOPT code
is devoted to handling the many files generated by various codes
inside. Thus at the end of a run the directory may be filled with
thousands of files. It is recommended that the user delete files
using 'rm *_opt*' as this command will remove the temporary
files which were generated. What remains are the minima found as the
optimizer executed. These minima are labeled by either their
function evaluation or generation number (optimizer dependant). In
all cases, the STELLOPT code produces a file called stellopt.<ext>
which is a text file containing information for each component of
the cost vector. The file has a generic format of:
```
STELLOPT VERSION
ITER NNN
NAME NROW NCOL HEADINGS
DATA
NAME NROW NCOL HEADINGS
DATA
ITER NNN
```
Here NNN is the
iteration number (function evaluation), NAME is the name of a cost
function variable, NROW is the number of row in the DATA, NCOL is
the number of columns in the DATA, HEADING is text decryping what
the columns mean, and DATA is the numeric data. Additionally, and
xvec.dat file and var\_lables file are generated. The xvec file
saves a copy of the optimization vectors evaluated and their
associated chi-squared. While the var_labels file contains a
description of each component of that vector.
